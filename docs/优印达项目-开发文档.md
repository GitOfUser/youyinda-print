## 一、项目概述

### 1.1 项目简介
优印达是一个集云打印和快递寄件于一体的微信小程序聚合平台，对标小猴云印交互与功能，提供线上云打印、低价快递寄件服务。

### 1.2 核心模式
纯平台化聚合，所有打印生产、快递揽派履约均对接第三方服务商API，平台仅做流量分发、价格封装、订单中转、盈利差价赚取。

### 1.3 盈利核心
第三方基础服务价叠加自定义可配置盈利规则，形成平台最终展示售价，所有规则支持后台热更新，禁止硬编码。

```

   ↓
3. 返回prepay_id等参数给小程序

4. 小程序调用wx.requestPayment发起支付
   ↓
5. 用户支付完成
   ↓
6. 微信异步通知支付结果
   ↓
7. 后端验证签名，更新订单状态
```

### 4.6 价格盈利引擎模块

#### 功能说明
- 打印价格计算
- 快递价格计算
- 盈利规则配置
- 平台售价计算

#### 核心文件
- `PriceController.java` - 价格查询接口
- `PriceCalculateUtil.java` - 价格计算工具
- `ProfitRule` - 盈利规则实体

#### 定价公式
```
平台售价 = 第三方基础价 × (1 + 盈利比例) + 固定盈利金额

或

平台售价 = 第三方基础价 + 固定盈利金额
```

#### 价格计算流程
```
1. 获取第三方基础价格
   ↓
2. 查询盈利规则配置
   ↓
3. 应用盈利规则计算平台售价
   ↓
4. 应用优惠券（如有）
   ↓
5. 返回最终价格
```

## 五、核心业务流程

### 5.1 打印业务流程

```
┌─────────────────────────────────────────────────────────────┐
│                        打印订单流程                            │
└─────────────────────────────────────────────────────────────┘

1. 用户进入打印页面
   ↓
2. 上传文件（PDF/Word/Excel/图片）
   ↓
3. 配置打印参数
   - 纸张类型（A4/A3/A5）
   - 色彩类型（黑白/彩色）
   - 单双面（单面/双面）
   - 打印份数
   - 装订方式
   ↓
4. 填写配送地址
   ↓
5. 系统计算价格
   - 查询打印基础价格
   - 应用盈利规则
   - 计算配送费
   ↓
6. 确认订单信息
   ↓
7. 调用微信支付
   ↓
8. 支付成功
   ↓
9. 创建订单记录
   - order_main（订单主表）
   - order_detail（订单明细表）
   ↓
10. 调用第三方打印服务商API下单
    ↓
11. 更新订单状态为"处理中"
    ↓
12. 服务商打印生产
    ↓
13. 服务商安排配送
    ↓
14. 更新订单状态为"配送中"
    ↓
15. 用户签收
    ↓
16. 更新订单状态为"已完成"
```

### 5.2 快递业务流程

```
┌─────────────────────────────────────────────────────────────┐
│                        快递订单流程                            │
└─────────────────────────────────────────────────────────────┘

1. 用户进入快递寄件页面

   ↓

   - 计算首重+续重费用
   ↓
7. 确认订单信息
   ↓

   ↓
9. 支付成功
   ↓
10. 创建订单记录
    - order_main（订单主表）

    ↓
11. 调用第三方快递服务商API下单
    ↓
12. 更新订单状态为"待揽收"
    ↓
13. 快递员上门揽件
    ↓
14. 更新订单状态为"运输中"
    ↓
15. 定时同步物流信息
    ↓
16. 快递送达
    ↓
17. 更新订单状态为"已完成"
```

### 7.3 统一响应类 R.java
```java
@Data
public class R<T> {
    private Integer code;
    private String message;
    private T data;

    public static <T> R<T> ok() {
        return ok(null);
    }

    public static <T> R<T> ok(T data) {
        R<T> r = new R<>();
        r.setCode(200);
        r.setMessage("success");
        r.setData(data);
        return r;
    }

    public static <T> R<T> fail(String message) {
        R<T> r = new R<>();
        r.setCode(500);
        r.setMessage(message);
        return r;
    }

    public static <T> R<T> fail(Integer code, String message) {
        R<T> r = new R<>();
        r.setCode(code);
        r.setMessage(message);
        return r;
    }
}
```

### 7.4 Controller 示例

@RestController

## 八、异常处理、日志规范

### 8.1 异常处理

#### 自定义业务异常
```java
@Getter
public class BusinessException extends RuntimeException {
    private Integer code;

    public BusinessException(String message) {
        super(message);
        this.code = 500;
    }

    public BusinessException(Integer code, String message) {
        super(message);
        this.code = code;
    }

    public BusinessException(ErrorCodeEnum errorCode) {
        super(errorCode.getMessage());
        this.code = errorCode.getCode();
    }
}
```

#### 错误码枚举
```java
@Getter
@AllArgsConstructor
public enum ErrorCodeEnum {
    SUCCESS(200, "成功"),
    PARAM_ERROR(400, "参数错误"),
    UNAUTHORIZED(401, "未登录"),
    FORBIDDEN(403, "无权限"),
    NOT_FOUND(404, "资源不存在"),
    SYSTEM_ERROR(500, "系统错误"),
    ORDER_NOT_FOUND(1001, "订单不存在"),
    ORDER_STATUS_ERROR(1002, "订单状态错误"),
    PAY_ERROR(1003, "支付失败"),
    PRINT_PROVIDER_ERROR(2001, "打印服务商异常"),
    EXPRESS_PROVIDER_ERROR(2002, "快递服务商异常");

    private final Integer code;
    private final String message;
}
```

#### 全局异常处理器
```java
@RestControllerAdvice
@Slf4j
public class GlobalExceptionHandler {

    @ExceptionHandler(BusinessException.class)
    public R<Void> handleBusinessException(BusinessException e) {
        log.warn("业务异常: {}", e.getMessage());
        return R.fail(e.getCode(), e.getMessage());
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public R<Void> handleValidationException(MethodArgumentNotValidException e) {
        String message = e.getBindingResult().getFieldError().getDefaultMessage();
        log.warn("参数校验异常: {}", message);
        return R.fail(400, message);
    }

    @ExceptionHandler(Exception.class)
    public R<Void> handleException(Exception e) {
        log.error("系统异常", e);
        return R.fail(500, "系统错误，请联系客服");
    }
}
```

### 8.2 日志规范

#### 日志级别使用
- **ERROR**: 系统错误、异常
- **WARN**: 业务警告、参数校验失败
- **INFO**: 重要业务流程、关键操作
- **DEBUG**: 调试信息、详细流程

#### 日志格式
```
%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{50} - %msg%n
```

#### 日志示例

@Service

    @Transactional(rollbackFor = Exception.class)

## 九、开发注意事项

### 9.1 代码规范
- 严格遵循阿里巴巴Java开发规范
- 所有类、方法、核心逻辑必须添加中文注释
- 命名采用驼峰命名法，见名知意
- 分层架构清晰：controller → service → serviceImpl → mapper → entity
- 核心业务流程必须添加@Transactional事务注解，rollbackFor = Exception.class

### 9.2 安全规范
- 所有接口必须做参数校验（使用@Valid）
- 敏感信息（手机号、身份证号）必须AES加密存储
- 配置文件中的密钥、密码禁止硬编码
- SQL注入防护：使用MyBatis-Plus，避免拼接SQL
- XSS防护：输出内容进行转义
- 接口鉴权：除登录接口外，所有接口需要Token

### 9.3 性能规范
- 合理使用Redis缓存，热点数据必须缓存
- 数据库查询必须添加索引，避免全表扫描
- 避免N+1查询，使用JOIN或批量查询
- 分页查询使用MyBatis-Plus分页插件
- 大文件上传使用分片上传
- 接口响应时间控制在500ms以内

### 9.4 业务规范
- 价格计算逻辑必须无漏洞，所有计价公式完全对用户透明
- 计算结果保留2位小数
- 第三方接口必须实现：配置化管理、超时重试、熔断降级、全链路日志追踪、异常兜底处理
- 订单状态流转必须有完整的日志记录
- 状态变更必须做前置校验，禁止非法状态跳转
- 盈利规则必须支持后台热更新，禁止硬编码

### 9.5 配置规范
- 所有第三方接口配置（url、appId、密钥、超时时间）必须抽离到application.yml配置文件
- 使用环境变量或配置中心管理生产环境配置
- 配置文件分环境：dev、test、prod
- 敏感配置使用Jasypt加密

### 9.6 Git规范
- 分支管理：master（生产）、develop（开发）、feature/*（功能）、hotfix/*（修复）
- 提交信息格式：[类型] 描述，如：[feat] 添加打印订单功能
- 提交类型：feat（新功能）、fix（修复）、docs（文档）、style（格式）、refactor（重构）、test（测试）、chore（构建）
- 代码合并前必须Code Review

### 9.7 测试规范
- 核心业务逻辑必须编写单元测试
- 接口测试使用Postman或Apifox
- 上线前必须完成全流程测试
- 测试用例覆盖正常场景、边界场景、异常场景
