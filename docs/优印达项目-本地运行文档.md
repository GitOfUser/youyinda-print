## 二、数据库初始化步骤

### 2.1 安装MySQL

#### Windows安装

1. 下载MySQL Installer
2. 运行安装程序，选择"Developer Default"
3. 设置root密码（建议：root123456）
4. 配置端口：3306
5. 完成安装

#### Mac安装（Homebrew）

```bash
brew install mysql
brew services start mysql
mysql_secure_installation
```

#### Linux安装

```bash
# CentOS
yum install -y mysql-community-server
systemctl start mysqld

# Ubuntu
apt-get install -y mysql-server
systemctl start mysql
```

### 2.2 创建数据库

#### 方式一：使用Navicat等工具

1. 连接MySQL（host: localhost, port: 3306, user: root, password: 你的密码）
2. 新建数据库，名称：youyinda
3. 字符集：utf8mb4
4. 排序规则：utf8mb4\_general\_ci

#### 方式二：使用命令行

```bash
# 登录MySQL
mysql -uroot -p

# 创建数据库
CREATE DATABASE youyinda DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

# 创建用户（可选）
CREATE USER 'youyinda'@'%' IDENTIFIED BY 'youyinda123';
GRANT ALL PRIVILEGES ON youyinda.* TO 'youyinda'@'%';
FLUSH PRIVILEGES;

EXIT;
```

### 2.3 导入初始化脚本

#### 方式一：使用Navicat

1. 打开youyinda数据库
2. 点击"运行SQL文件"
3. 选择 `youyinda-backend/src/main/resources/init.sql`
4. 点击开始，等待导入完成

#### 方式二：使用命令行

```bash
# 进入项目目录
cd d:\Projects\youyinda-print\youyinda-backend\src\main\resources

# 导入脚本
mysql -uroot -p youyinda < init.sql
```

#### 方式三：使用IDEA Database工具

1. 打开IDEA
2. 右侧打开Database面板
3. 点击"+" → Data Source → MySQL
4. 填写连接信息：
   - Host: localhost
   - Port: 3306
   - Database: youyinda
   - User: root
   - Password: 你的密码
5. 点击Test Connection测试连接
6. 连接成功后，右键数据库 → SQL Scripts → Run SQL Script
7. 选择init.sql文件执行

### 2.4 验证数据库

```sql
-- 查看表
USE youyinda;
SHOW TABLES;

-- 应该看到以下表：
-- user_info, user_address, order_main, order_detail, order_logistics
-- print_base_price, express_base_price, express_company
-- coupon_info, user_coupon, third_api_config, sys_config
-- admin_user, sys_operation_log
```

```yaml

  servlet:
    context-path: /api          # 接口路径前缀

    password: root123456                    # 改为你的MySQL密码
    druid:
      initial-size: 5
      min-idle: 5
      max-active: 20
      max-wait: 60000
      time-between-eviction-runs-millis: 60000
      min-evictable-idle-time-millis: 300000
      validation-query: SELECT 1
      test-while-idle: true
      test-on-borrow: false
      test-on-return: false
      filters: stat,wall
      web-stat-filter:
        enabled: true
        url-pattern: /*

      stat-view-servlet:
        enabled: true
        url-pattern: /druid/*

  redis:
    host: localhost                    # Redis地址
    port: 6379                         # Redis端口
    password:                          # Redis密码（如有）
    database: 0
    timeout: 10000ms
    lettuce:
      pool:

        max-wait: -1ms
        max-idle: 8
        min-idle: 0

    time-zone: GMT+8
    date-format: yyyy-MM-dd HH:mm:ss

# MyBatis-Plus配置
mybatis-plus:
  mapper-locations: classpath*:/mapper/**/*.xml
  type-aliases-package: com.youyinda.entity
  configuration:
    map-underscore-to-camel-case: true
    cache-enabled: false

        max-attempts: 3

          - java.io.IOException
```

## 五、后端启动步骤

### 5.1 方式一：使用IDEA启动（推荐）

#### 步骤1：导入项目

1. 打开IntelliJ IDEA
2. 选择 File → Open
3. 选择 `d:\Projects\youyinda-print\youyinda-backend` 目录
4. 等待Maven依赖下载完成（右下角进度条）

#### 步骤2：配置Maven

1. 打开 File → Settings → Build, Execution, Deployment → Build Tools → Maven
2. 检查Maven home directory是否正确
3. 检查User settings file是否正确
4. 点击Apply → OK

#### 步骤3：刷新Maven

1. 右侧打开Maven面板
2. 点击刷新按钮（Reload All Maven Projects）
3. 等待依赖下载完成

#### 步骤4：启动项目

1. 找到启动类：`src/main/java/com/youyinda/YouyindaApplication.java`
2. 右键点击 → Run 'YouyindaApplication'
3. 或点击类名左侧的绿色三角形按钮

#### 步骤5：验证启动成功

看到以下日志表示启动成功：

```
Started YouyindaApplication in X.XXX seconds
```

访问测试：

- 健康检查：<http://localhost:8080/api/actuator/health>
- Druid监控：<http://localhost:8080/api/druid/>

### 5.2 方式二：使用Maven命令启动

#### 步骤1：编译项目

```bash
cd d:\Projects\youyinda-print\youyinda-backend
mvn clean compile
```

#### 步骤2：启动项目

```bash
mvn spring-boot:run
```

### 5.3 方式三：打包后启动

#### 步骤1：打包

```bash
cd d:\Projects\youyinda-print\youyinda-backend
mvn clean package -DskipTests
```

#### 步骤2：运行JAR包

```bash
cd target
java -jar youyinda-backend.jar
```

或指定配置文件：

```bash
java -jar youyinda-backend.jar --spring.profiles.active=dev
```

## 六、小程序导入与启动步骤

### 6.1 准备工作

#### 获取小程序AppID

3. 配置uploadFile合法域名：<https://your-api-domain.com>
4. 配置downloadFile合法域名：<https://your-api-domain.com>

**注意**：本地调试时可以在开发者工具中关闭域名校验。

### 6.2 导入小程序项目

3. 使用微信扫码登录

1. 点击「+」号或「导入项目」

   - 后端服务：不使用云服务

## 八、常见启动问题与解决方案

### 8.1 后端启动问题

#### 问题1：端口被占用

**错误信息**：

```
Web server failed to start. Port 8080 was already in use.
```

**解决方案**：

```bash
# Windows查找占用8080端口的进程
netstat -ano | findstr "8080"

# 结束进程（PID替换为实际的进程ID）
taskkill /F /PID <PID>

# 或修改application.yml中的端口号
server:
  port: 8081
```

#### 问题2：数据库连接失败

**错误信息**：

```
Communications link failure
Access denied for user 'root'@'localhost'
```

**解决方案**：

1. 检查MySQL是否启动
2. 检查application.yml中的数据库地址、用户名、密码是否正确
3. 检查数据库是否已创建
4. 检查MySQL用户权限

#### 问题3：Redis连接失败

**错误信息**：

```
Unable to connect to Redis
```

**解决方案**：

1. 检查Redis是否启动
2. 检查Redis地址、端口、密码是否正确
3. Windows下检查Memurai是否启动

#### 问题4：Maven依赖下载失败

**解决方案**：

1. 配置Maven镜像源（阿里云）

```xml
<!-- 在settings.xml中添加 -->
<mirror>
  <id>aliyunmaven</id>
  <mirrorOf>*</mirrorOf>
  <name>阿里云公共仓库</name>
  <url>https://maven.aliyun.com/repository/public</url>
</mirror>
```

1. 在IDEA中刷新Maven
2. 检查网络连接

#### 问题5：JDK版本不匹配

**错误信息**：

```
Unsupported class file major version 52
```

**解决方案**：

1. 检查IDEA的Project SDK：File → Project Structure → Project
2. 设置SDK为1.8
3. 检查Modules的Language level：File → Project Structure → Modules
4. 设置为8 - Lambdas, type annotations etc.

### 8.2 小程序启动问题

#### 问题1：AppID未配置

**错误信息**：

```
appid 不能为空
```

**解决方案**：

1. 打开project.config.json
2. 填入正确的小程序AppID
3. 或选择「测试号」

#### 问题2：接口请求失败

**错误信息**：

```
request:fail url not in domain list
```

**解决方案**：

1. 打开微信开发者工具
2. 点击右上角「详情」
3. 勾选「不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书」

#### 问题3：编译报错

**解决方案**：

1. 检查代码语法错误
2. 查看Console中的错误信息
3. 清除缓存：工具 → 清除缓存 → 全部清除
4. 重新编译

### 8.3 数据库问题

#### 问题1：导入SQL脚本失败

**解决方案**：

1. 确保数据库已创建
2. 确保字符集为utf8mb4
3. 逐段执行SQL，定位错误

#### 问题2：中文乱码

**解决方案**：

1. 确保数据库字符集为utf8mb4
2. JDBC URL添加参数：useUnicode=true\&characterEncoding=utf8
3. 确保表的字符集也是utf8mb4
