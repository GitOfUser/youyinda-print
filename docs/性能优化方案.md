## 一、慢SQL优化

### 1.1 当前慢SQL分析

#### OrderMain表查询优化

**问题SQL**：
```sql
-- 订单列表查询，没有合理索引
SELECT * FROM order_main 
WHERE user_id = #{userId} 
AND order_type = #{orderType}
AND status = #{status}
ORDER BY create_time DESC
LIMIT #{offset}, #{pageSize};
```

**优化方案**：
```sql
-- 添加复合索引
ALTER TABLE order_main ADD INDEX idx_user_type_status_create (user_id, order_type, status, create_time);

-- 优化查询语句，只查询需要的字段
SELECT id, order_no, order_type, status, total_amount, create_time 
FROM order_main 
WHERE user_id = #{userId} 
AND (order_type = #{orderType} OR #{orderType} IS NULL)
AND (status = #{status} OR #{status} IS NULL)
ORDER BY create_time DESC
LIMIT #{offset}, #{pageSize};
```

#### 订单详情查询优化

**问题SQL**：
```sql
-- 多次关联查询
SELECT * FROM order_main 
LEFT JOIN order_detail ON order_main.id = order_detail.order_id
LEFT JOIN order_logistics ON order_main.id = order_logistics.order_id
WHERE order_main.id = #{orderId};
```

**优化方案**：
```sql
-- 分两次查询，利用缓存
-- 第一次：查询订单主表
SELECT * FROM order_main WHERE id = #{orderId};

-- 第二次：查询订单明细（如果有）
SELECT * FROM order_detail WHERE order_id = #{orderId};

-- 第三次：查询物流信息（快递订单）
SELECT * FROM order_logistics WHERE order_id = #{orderId};
```

### 1.2 索引优化清单

| 表名 | 索引名 | 索引字段 | 用途 |
|------|--------|---------|------|
| order_main | idx_user_type_status_create | user_id, order_type, status, create_time | 订单列表查询 |
| order_main | idx_order_no | order_no | 订单号查询 |
| order_detail | idx_order_id | order_id | 订单明细查询 |
| order_logistics | idx_order_id | order_id | 物流信息查询 |
| user_address | idx_user_id_default | user_id, is_default | 用户地址查询 |
| user_info | idx_openid | openid | 微信用户查询 |
| coupon_info | idx_user_status | user_id, status | 用户优惠券查询 |
| print_base_price | idx_paper_color | paper_size, color_type | 打印价格查询 |
| express_base_price | idx_from_to_weight | from_province, to_province, min_weight, max_weight | 快递价格查询 |

### 1.3 SQL编写规范

1. **避免SELECT ***：只查询需要的字段
2. **使用合理的索引**：确保WHERE、ORDER BY、JOIN字段有索引
3. **分页查询优化**：大偏移量分页使用延迟关联
4. **避免在WHERE子句中使用函数**：会导致索引失效
5. **合理使用UNION ALL**：比UNION效率更高
6. **批量操作**：使用批量插入、批量更新

## 二、接口响应优化

### 2.1 接口响应时间目标

| 接口类型 | 响应时间目标 | P99响应时间 |
|---------|-------------|------------|
| 静态数据接口 | < 100ms | < 200ms |
| 简单查询接口 | < 200ms | < 500ms |
| 复杂查询接口 | < 500ms | < 1s |
| 订单创建接口 | < 1s | < 2s |
| 支付接口 | < 2s | < 5s |

### 2.2 接口优化措施

#### 2.2.1 异步处理

**需要异步化的操作**：
- 订单创建成功后的操作日志记录
- 支付成功后的消息推送
- 物流信息同步（定时任务）
- 数据统计更新

**实现方式**：
```java
@Async
public void recordOrderLog(OrderLog orderLog) {
    orderLogMapper.insert(orderLog);
}

@Async
public void syncExpressTrack(Long orderId) {
    expressTrackService.syncTrack(orderId);
}
```

#### 2.2.2 接口合并与拆分

**合并接口**：
- 将多个相关的查询合并为一个接口
- 减少网络请求次数

**拆分接口**：
- 将大接口拆分为多个小接口
- 支持按需加载
- 首页数据使用懒加载

#### 2.2.3 数据传输优化

- **使用DTO**：只返回前端需要的字段
- **日期格式化**：后端格式化好日期，前端直接展示
- **分页响应**：大列表必须分页
- **数据压缩**：开启Gzip压缩

## 三、缓存优化

### 3.1 缓存策略

| 数据类型 | 缓存方式 | 缓存时间 | 说明 |
|---------|---------|---------|------|
| 打印基础价格 | Redis | 1小时 | 变化不频繁 |
| 快递基础价格 | Redis | 1小时 | 变化不频繁 |
| 快递公司列表 | Redis | 24小时 | 基本不变 |
| 用户信息 | Redis | 30分钟 | 登录用户信息 |
| 热门数据 | Redis | 5分钟 | 热点数据 |
| 接口幂等性 | Redis | 5分钟 | 防止重复提交 |

### 3.2 Redis缓存实现

#### 打印价格缓存

```java
@Service
public class PrintPriceService {
    
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    private static final String PRINT_PRICE_KEY = "print:price:";
    
    public PrintBasePrice getPrintPrice(String paperSize, Integer colorType) {
        String key = PRINT_PRICE_KEY + paperSize + ":" + colorType;
        
        PrintBasePrice price = (PrintBasePrice) redisTemplate.opsForValue().get(key);
        if (price != null) {
            return price;
        }
        
        price = printBasePriceMapper.selectOne(
            new LambdaQueryWrapper<PrintBasePrice>()
                .eq(PrintBasePrice::getPaperSize, paperSize)
                .eq(PrintBasePrice::getColorType, colorType)
        );
        
        if (price != null) {
            redisTemplate.opsForValue().set(key, price, 1, TimeUnit.HOURS);
        }
        
        return price;
    }
}
```

#### 缓存预热

```java
@Component
public class CacheWarmupRunner implements ApplicationRunner {
    
    @Autowired
    private PrintBasePriceService printBasePriceService;
    
    @Autowired
    private ExpressBasePriceService expressBasePriceService;
    
    @Autowired
    private ExpressCompanyService expressCompanyService;
    
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    @Override
    public void run(ApplicationArguments args) {
        warmupPrintPrice();
        warmupExpressPrice();
        warmupExpressCompany();
    }
    
    private void warmupPrintPrice() {
        List<PrintBasePrice> list = printBasePriceService.list();
        list.forEach(price -> {
            String key = "print:price:" + price.getPaperSize() + ":" + price.getColorType();
            redisTemplate.opsForValue().set(key, price, 1, TimeUnit.HOURS);
        });
    }
}
```

### 3.3 缓存更新策略

- **主动更新**：数据变更时更新缓存
- **过期失效**：设置合理的过期时间
- **缓存击穿保护**：使用互斥锁
- **缓存雪崩保护**：随机过期时间
- **缓存穿透保护**：布隆过滤器或空值缓存

使用HikariCP连接池，配置如下：

```yaml
spring:
  datasource:
    type: com.zaxxer.hikari.HikariDataSource
    hikari:
      auto-commit: true
      connection-timeout: 30000
      idle-timeout: 600000
      max-lifetime: 1800000
      maximum-pool-size: 20
      minimum-idle: 5
      pool-name: YouyindaHikariCP
```

### 5.2 JVM参数优化

```bash
java -jar youyinda.jar \
  -Xms512m \
  -Xmx1024m \
  -XX:+UseG1GC \
  -XX:MaxGCPauseMillis=200 \
  -XX:+HeapDumpOnOutOfMemoryError \
  -XX:HeapDumpPath=/logs/heapdump.hprof \
  -Dfile.encoding=UTF-8
```

### 5.3 Nginx配置优化

```nginx
server {
    listen 80;
    server_name api.youyinda.com;
    
    # Gzip压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript application/x-javascript application/xml+rss application/json application/javascript;
    
    # 限流
    limit_req_zone $binary_remote_addr zone=api_limit:10m rate=10r/s;
    
    location /api {
        limit_req zone=api_limit burst=20 nodelay;
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        
        # 超时设置
        proxy_connect_timeout 5s;
        proxy_send_timeout 30s;
        proxy_read_timeout 30s;
    }
}
```
