# 优印达运维监控方案

## 一、监控架构

### 1.1 整体架构
```
┌─────────────────────────────────────────────────────────────┐
│                         监控系统                              │
├──────────────┬──────────────┬──────────────┬───────────────┤
│  应用监控    │  第三方监控  │  系统监控    │  告警系统      │
└──────────────┴──────────────┴──────────────┴───────────────┘
```

### 1.2 技术选型
- **应用监控**: Spring Boot Actuator + Micrometer + Prometheus + Grafana
- **日志监控**: ELK Stack (Elasticsearch + Logstash + Kibana)
- **系统监控**: Zabbix / Node Exporter
- **告警**: Alertmanager + 钉钉/企业微信群机器人
- **APM**: SkyWalking (可选)

### 3.2 健康检查实现

#### 健康检查定时任务
```java
@Component
@Slf4j
public class ThirdPartyHealthCheckTask {

    @Autowired
    private PrintProviderFeignClient printProviderFeignClient;

    @Autowired
    private ExpressProviderFeignClient expressProviderFeignClient;

    @Autowired
    private RedisTemplate<String, Object> redisTemplate;

    private static final String HEALTH_KEY_PREFIX = "health:third-party:";

    /**
     * 打印服务商健康检查 - 每1分钟执行
     */
    @Scheduled(fixedRate = 60 * 1000)
    public void checkPrintProviderHealth() {
        HealthCheckResult result = new HealthCheckResult();
        result.setServiceName("print-provider");
        result.setCheckTime(new Date());

        try {
            long start = System.currentTimeMillis();
            printProviderFeignClient.healthCheck();
            long cost = System.currentTimeMillis() - start;

            result.setStatus("UP");
            result.setResponseTime(cost);
            result.setMessage("Health check passed");

            // 清除连续失败计数
            redisTemplate.delete(HEALTH_KEY_PREFIX + "print-provider:fail-count");
        } catch (Exception e) {
            result.setStatus("DOWN");
            result.setMessage(e.getMessage());

            // 增加连续失败计数
            String failKey = HEALTH_KEY_PREFIX + "print-provider:fail-count";
            Long failCount = redisTemplate.opsForValue().increment(failKey);
            redisTemplate.expire(failKey, 10, TimeUnit.MINUTES);

            result.setFailCount(failCount != null ? failCount.intValue() : 1);

            log.error("打印服务商健康检查失败: {}", e.getMessage());
        }

        // 保存检查结果
        redisTemplate.opsForValue().set(HEALTH_KEY_PREFIX + "print-provider", result, 10, TimeUnit.MINUTES);
    }

    /**
     * 快递服务商健康检查 - 每1分钟执行
     */
    @Scheduled(fixedRate = 60 * 1000)
    public void checkExpressProviderHealth() {
        // 类似打印服务商实现
    }

    /**
     * 微信支付健康检查 - 每2分钟执行
     */
    @Scheduled(fixedRate = 2 * 60 * 1000)
    public void checkWxPayHealth() {
        // 微信支付健康检查实现
    }
}
```

### 4.2 关键日志格式

#### 订单日志
```java
@Aspect
@Component
@Slf4j
public class OrderLogAspect {

    @Pointcut("@annotation(com.youyinda.annotation.OrderLog)")
    public void orderLogPointcut() {}

    @Around("orderLogPointcut()")
    public Object logOrder(ProceedingJoinPoint joinPoint) throws Throwable {
        long start = System.currentTimeMillis();
        String methodName = joinPoint.getSignature().getName();
        Object[] args = joinPoint.getArgs();

        log.info("订单操作开始: method={}, args={}", methodName, args);

        Object result = null;
        try {
            result = joinPoint.proceed();
            long cost = System.currentTimeMillis() - start;
            log.info("订单操作成功: method={}, cost={}ms, result={}", methodName, cost, result);
            return result;
        } catch (Exception e) {
            long cost = System.currentTimeMillis() - start;
            log.error("订单操作失败: method={}, cost={}ms, error={}", methodName, cost, e.getMessage(), e);
            throw e;
        }
    }
}
```

```java
    public Object logThirdPartyApi(ProceedingJoinPoint joinPoint) throws Throwable {

        String methodName = joinPoint.getSignature().getName();
        Object[] args = joinPoint.getArgs();

        long start = System.currentTimeMillis();
        log.info("第三方接口调用开始: service={}, method={}, args={}", className, methodName, args);

            return result;
        } catch (Exception e) {
            long cost = System.currentTimeMillis() - start;
            log.error("第三方接口调用失败: service={}, method={}, cost={}ms, error={}",
```

### 5.4 钉钉告警通知实现

```java
@RestController
@RequestMapping("/webhook")
@Slf4j
public class AlertWebhookController {

    @Value("${dingtalk.webhook.url}")
    private String dingtalkWebhookUrl;

    @Value("${dingtalk.webhook.secret}")
    private String dingtalkSecret;

    /**
     * 接收Alertmanager告警并转发到钉钉
     */
    @PostMapping("/dingtalk")
    public void handleAlert(@RequestBody AlertManagerMessage message) {
        for (Alert alert : message.getAlerts()) {
            String markdownContent = buildMarkdownContent(alert);
            sendDingtalkMessage(markdownContent, alert.getLabels().get("severity"));
        }
    }

    private String buildMarkdownContent(Alert alert) {
        StringBuilder sb = new StringBuilder();
        String status = alert.getStatus().equals("firing") ? "🚨 告警触发" : "✅ 告警恢复";
        sb.append("### ").append(status).append("\n\n");
        sb.append("**告警名称**: ").append(alert.getLabels().get("alertname")).append("\n\n");
        sb.append("**告警级别**: ").append(alert.getLabels().get("severity")).append("\n\n");
        sb.append("**告警时间**: ").append(alert.getStartsAt()).append("\n\n");
        sb.append("**告警描述**: ").append(alert.getAnnotations().get("description")).append("\n\n");
        sb.append("**实例**: ").append(alert.getLabels().get("instance")).append("\n");
        return sb.toString();
    }

    private void sendDingtalkMessage(String markdownContent, String severity) {
        try {
            long timestamp = System.currentTimeMillis();
            String sign = generateSign(timestamp);

            String url = dingtalkWebhookUrl + "&timestamp=" + timestamp + "&sign=" + sign;

            DingtalkMessage message = new DingtalkMessage();
            message.setMsgtype("markdown");
            Markdown markdown = new Markdown();
            markdown.setTitle("优印达告警 - " + severity);
            markdown.setText(markdownContent);
            message.setMarkdown(markdown);

            // 发送HTTP请求到钉钉
            RestTemplate restTemplate = new RestTemplate();
            restTemplate.postForObject(url, message, String.class);

            log.info("钉钉告警发送成功: severity={}", severity);
        } catch (Exception e) {
            log.error("钉钉告警发送失败", e);
        }
    }

    private String generateSign(long timestamp) throws Exception {
        String stringToSign = timestamp + "\n" + dingtalkSecret;
        Mac mac = Mac.getInstance("HmacSHA256");
        mac.init(new SecretKeySpec(dingtalkSecret.getBytes("UTF-8"), "HmacSHA256"));
        byte[] signData = mac.doFinal(stringToSign.getBytes("UTF-8"));
        return URLEncoder.encode(new String(Base64.getEncoder().encode(signData)), "UTF-8");
    }
}
```

## 九、应急响应流程

### 9.1 告警分级响应

#### P0 - 紧急 (Critical)
- **定义**: 系统完全不可用，影响所有用户
- **响应时间**: 立即响应，15分钟内开始处理
- **处理人员**: 技术负责人 + 运维 + 核心开发
- **升级流程**: 1小时未解决升级CTO

**触发场景**:
- 服务完全宕机
- 数据库宕机
- 支付接口完全不可用
- 数据泄露

#### P1 - 高 (High)
- **定义**: 核心功能不可用，影响大量用户
- **响应时间**: 30分钟内响应，1小时内开始处理
- **处理人员**: 运维 + 对应模块开发
- **升级流程**: 2小时未解决升级技术负责人

**触发场景**:
- 打印/快递订单无法创建
- 支付成功率低于80%
- 接口错误率高于10%

#### P2 - 中 (Medium)
- **定义**: 次要功能异常，影响部分用户体验
- **响应时间**: 4小时内响应，工作日内处理
- **处理人员**: 对应模块开发

**触发场景**:
- 某个第三方服务商接口异常
- 非核心页面加载慢
- 告警阈值触发

#### P3 - 低 (Low)
- **定义**: 轻微问题，不影响核心功能
- **响应时间**: 下一工作日处理
- **处理人员**: 对应模块开发

**触发场景**:
- 日志告警
- 性能轻微下降
- 功能建议

### 9.2 故障处理流程

```
1. 告警接收
   ↓
2. 告警确认（确认是否误报）
   ↓
3. 故障定级（P0/P1/P2/P3）
   ↓
4. 通知相关人员
   ↓
5. 故障排查
   - 查看监控指标
   - 查看错误日志
   - 检查系统状态
   - 复现问题
   ↓
6. 故障定位
   ↓
7. 制定恢复方案
   ↓
8. 执行恢复操作
   - 回滚代码
   - 重启服务
   - 切换备机
   - 限流降级
   ↓
9. 验证恢复
   ↓
10. 故障记录与复盘
    - 故障时间线
    - 根因分析
    - 改进措施
    - 经验总结
```

## 十一、成本优化建议

### 11.1 服务器资源优化
- 根据实际负载调整服务器配置

- 定期归档历史数据

- 使用读写分离
- 考虑使用云数据库托管服务

- 使用多级缓存
- 监控缓存命中率，及时调整缓存策略
